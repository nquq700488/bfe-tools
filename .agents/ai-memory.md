# AI 共享记忆规范

> `AI_MEMORY.md` 是"AI 首读项目认知入口"，用于快速建立项目心智模型；
> 同时沉淀跨任务、可复用、长期有效的架构事实与经验规则。

---

## 目标

- 让任意 AI 读完后能快速理解项目架构、关键链路和高风险区域
- 避免同类问题跨会话重复排查
- 沉淀可复用的工程约束与边界
- 保持内容高信噪比，避免任务流水污染

---

## 写入触发条件（命中任一即写）

- 形成了跨模块通用规则（可约束后续实现/审查/测试）
- 确认了非显而易见的根因，且修复已验证有效
- 确认了长期有效的架构事实或技术边界（如框架限制、平台兼容性约束）
- 环境与工具链基线发生稳定变更（Node/包管理/构建关键配置）
- 用户明确要求"以后都按这个做"且规则可泛化

---

## 不写入条件

- 单次任务流水、临时调试过程
- 仅当前文件有效的局部实现细节
- 未验证的猜测、待确认方案
- 可在 `TASK_MEMORY.md` 追溯的短期任务状态

说明：

- 可以写"项目认知必需"的中层信息（如架构地图、关键链路、风险区域）
- 但不应写到具体函数实现步骤或大段代码片段

---

## 写入时机

- 决策确认或根因验证后立即写入
- 长流程任务在收尾阶段做一次去重与归并
- 上下文压缩前，先补齐本轮新增的长期经验

---

## 写入格式（最小模板）

```markdown
- **[分类]** 规则结论。必要时补一句边界条件或失败后果（可选）
```

分类建议：`数据`、`并发`、`状态`、`交互`、`布局`、`日志`、`错误处理`、`配置`、`性能`、`测试`。

---

## 推荐结构（入口型）

1. **项目概览**（业务 + 技术栈 + 入口模块）
2. **架构地图**（页面层 / 状态层 / 核心领域层 / 基础设施层）
3. **关键业务链路**（核心流程的描述和风险点）
4. **高风险区域**（改前优先补读的区域列表）
5. **长期有效规则**（跨任务复用的工程约束）

---

## 写入流程

1. 查重：先检索 `AI_MEMORY.md` 是否已有同类规则
2. 归类：写入已有章节；无对应章节再新增
3. 精炼：每条规则优先 1 句结论；必要时补 1 句边界/后果，禁止任务复盘叙事
4. 校验：确保"可执行、可复用、已验证"

---

## 责任分工

- Developer/Reviewer/Tester：发现可沉淀经验时立即提出并更新
- Orchestrator：交付前确认本轮是否命中写入条件，避免漏记

---

## 与 TASK MEMORY 的边界

- `TASK_MEMORY.md`：记录"这次任务怎么做、为什么这么做"
- `AI_MEMORY.md`：记录"以后类似问题都该怎么做"

---

## 示例（AI_MEMORY.md 长成什么样）

以下是一个前端项目的精简示例，展示推荐结构和写法风格：

```markdown
# AI_MEMORY.md — 项目共享认知入口

## 项目概览

- 业务：移动端 H5 作业批改（老师对学生拍照作业进行批注、评分、提交）
- 技术栈：Vite + Vue3 + TypeScript + Pinia + Vant + Fabric.js
- 代码主入口：`src/pages/lecture-note.vue`

## 架构地图

### 页面层
- `lecture-note.vue` 负责整页装配、初始化、Native 生命周期联动

### 业务状态层（Pinia）
- `correction.ts` 聚合 store，组合作业加载、图片预加载、痕迹读写等子模块

### 画布层（Fabric）
- `ImageCanvas/index.vue` 是画布交互中枢

## 关键业务链路

### 链路 A：作业加载与切换
- 列表加载 → 详情加载 → 状态恢复 → 图片切换 → 画布同步
- 风险点：同步修改多个状态字段时，watcher 执行顺序与旧值语义容易错位

## 长期有效规则

- **[数据]** 读取数据前先确认写入路径，禁止依赖"从未写入过"的字段
- **[并发]** 并发池任务必须在自身 `finally` 自清理
- **[状态]** 保存/恢复必须语义对称，存在归一化就必须有逆操作
```
