# 执行模式判断规范

> 本文件只回答两个问题：
>
> 1. 当前任务走完整模式还是轻量模式
> 2. 进入编码前是并行还是串行

---

## 模式判断流程

### 步骤 1：4 项指标评估

| 指标 | 判定 |
|------|------|
| 是否跨职责边界 | 是/否 |
| 是否改变公共契约 | 是/否 |
| 是否改变数据流 / 状态流 | 是/否 |
| 是否存在独立决策面 | 是/否 |

**辅助观察项（不计入命中数）**：

- 预计涉及文件数
- 是否为机械性改动
- 是否位于核心链路

### 步骤 2：降级条件检查

满足以下**全部三条**，可降级为轻量模式：

- 单因修改（所有改动源于同一个根因）
- 无新增业务逻辑
- 影响面可控

> ⚠️ 不确定是否满足降级条件时，默认**不满足**，不允许自我宽松判定。

### 步骤 3：模式确定

- 命中 0 项 → **【轻量模式】**：直接编码
- 命中 1 项：
  - 机械性改动、无独立决策、影响面可控 → **【轻量模式】**
  - 涉及核心链路、高回归风险或边界不清 → **【完整模式】**
- 命中 ≥ 2 项 → **【完整模式】**：Plan → Dev → Review → Test

> 多模型是否介入，不在本文件判断，见 `collaboration.md`。

### 步骤 4：并行执行预判（初步）

| 预判维度 | 评估内容 |
|----------|----------|
| 文件独立性 | 各文件修改是否相互独立（无依赖、无冲突） |
| 类型影响面 | 是否涉及类型定义改动（有则大概率需串行） |
| 业务复杂度 | 是否涉及复杂状态流转或数据流变更 |

预判结论：**🔮 预判可并行** 或 **🔮 预判需串行**

> ⚠️ 此预判仅为初步判断，**必须在读取所有文件后进行正式分析**。

### 步骤 5：模式确认

向用户展示判定结果。完整模式、不确定或高风险时等待确认后继续；轻量模式且用户未要求"做之前先问我"时，同步判断后直接执行。

输出格式：

```md
## 🔍 模式判断结果

| 指标 | 判定 | 说明 |
|------|------|------|
| 是否跨职责边界 | ✅/❌ | |
| 是否改变公共契约 | ✅/❌ | |
| 是否改变数据流 / 状态流 | ✅/❌ | |
| 是否存在独立决策面 | ✅/❌ | |

命中 X/4 项

辅助观察：预计 X 个文件 · 机械性改动 ✅/❌ · 核心链路 ✅/❌

降级条件：单因修改 ✅/❌ · 无新增逻辑 ✅/❌ · 影响面可控 ✅/❌

**建议模式：【完整模式】/【轻量模式】**

并行预判：文件独立性 高/中/低 · 类型影响面 大/中/小 · 业务复杂度 高/中/低
🔮 预判可并行 / 🔮 预判需串行（需读取文件后正式确认）

**下一步**：【等待确认 / 轻量模式直接执行】
```

## 步骤 6：并行执行分析（正式）

### 触发时机

| 模式 | 触发时机 |
|------|----------|
| 完整模式 | 方案确认后、实施前 |
| 轻量模式 | 开始编码前，读取所有文件后 |

### 前置要求

1. ✅ 读取所有涉及文件，不能凭印象
2. ✅ 扫描类型定义，确认是否已有可复用接口
3. ✅ 记录类型缺口，必要时先补前置项

### 并行评估 4 项指标

| 指标 | 判定标准 |
|------|----------|
| 子任务间依赖 | 无依赖 → ✅ 可并行 |
| 文件冲突风险 | 无冲突 → ✅ 可并行 |
| 类型一致性 | 不涉及或类型已存在 → ✅ 可并行 |
| 独立可验证 | 可独立验证 → ✅ 可并行 |

- **满足 ≥ 3 项**：【✅ 可并行】→ 单响应内并行修改
- **满足 ≤ 2 项**：【❌ 串行实施】→ 分步骤修改，每步验证后再进行下一步

### 输出格式

```md
## 📊 并行执行分析

### 文件依赖分析
| 文件 | 修改内容 | 依赖项 | 被依赖项 |
|------|---------|--------|---------|

### 批次划分
- Batch 1：[文件] — 必须先完成，其他文件依赖
- Batch 2：[文件] — 可并行
- ...

**执行策略**：【串行实施（分批次）】或【部分并行（Batch N 内并行）】
```

## 强制提示词

| 提示词 | 效果 |
|--------|------|
| 「完整模式」 | 强制 Plan → Dev → Review → Test 流程 |
| 「先评估」 | 先执行模式判断，确认后再决定流程 |
| 「按规范执行」 | 严格执行 AGENTS.md 所有检查清单 |
| 「硬性停止点」 | 每个关键节点暂停，等待用户确认 |
| 「先规划方案给我看」 | 强制 Planner 介入 |

---

## 关键节点暂停

| 节点 | 暂停内容 |
|------|----------|
| 模式判断后 | 完整模式/不确定/高风险时询问模式；轻量模式同步后继续 |
| 并行分析后 | 展示批次划分；仅高风险划分或用户要求确认时暂停 |
| 方案完成后 | 展示方案摘要，询问是否确认 |
| 审查未通过 | 展示问题列表，询问是否修复 |
| 测试失败后 | 展示失败用例，询问是否修复 |

---

## 快速指令

| 指令 | 作用 |
|------|------|
| `/plan` | 强制进入规划阶段 |
| `/dev` | 跳过规划，直接进入开发（仅轻量模式） |
| `/review` | 对已完成代码进行审查 |
| `/test` | 执行测试验证 |
| `/full` | 强制使用完整模式 |
| `/light` | 强制使用轻量模式 |
