# Hook: 会话结束（Session End）

> **模式**：`manual-check`（工具不支持真 hook 自动触发时，Agent 手动执行本清单）
>
> 触发时机：任务完成、用户主动结束会话、或上下文压缩前。
> 作用：自动保存会话摘要，防止关键决策和进度丢失。

---

## 强制收尾清单

### 1. 记忆落盘检查

| 检查项 | 条件 | 动作 |
|--------|------|------|
| TASK_MEMORY.md | 长任务 / 多轮决策 / 存在未完成事项 | 更新任务状态和关键决策 |
| AI_MEMORY.md | 发现新的架构坑位 / 长期有效规则 / 工具链问题 | 追加到对应分类 |

### 2. 重复模式回顾（持续学习信号收集）

每次收尾必须执行，检查当前任务是否为持续学习提供了新信号。详细规则见 `continuous-learning.md`「观察记录」章节。

**步骤**：

1. 读取 `AI_MEMORY.md`「观察中模式」章节（如不存在则跳过）
2. 遍历已有观察项，判断本次任务是否命中同一模式 → 命中则 N+1
3. 检查本次任务是否出现新信号（同类 bug / 同类任务 / 同一坑位 / 规范绕过 / 技术栈变更）→ 新增观察项
4. 输出模式回顾：

   ```text
   【模式回顾】
   已有观察项：N 个
   本次命中：[条目] → N：X → X+1（[是否达阈值]）
   本次新增：[条目]
   下次行动：[无 / 观察项 xxx 已达阈值，建议进入持续学习进化流程]
   ```

5. 如 `AI_MEMORY.md` 中尚无「观察中模式」章节，检查是否需要新建

### 3. 代码清理检查

- [ ] 无遗留 `console.log`
- [ ] 无遗留 `TODO` 占位符（未完成的必须显式说明）
- [ ] 无临时调试代码
- [ ] 无未使用的导入/变量
- [ ] 无 mock 数据混入生产代码

### 4. 验证结果确认

- [ ] lint 无 error
- [ ] 类型检查通过
- [ ] 必要测试已运行并通过（或显式标注未测试原因）
- [ ] 改动范围与用户要求一致（无越界修改）

### 5. 规范执行自检

- [ ] 已完成任务路由
- [ ] 已读取必要规范
- [ ] 已按选定 skill 流程执行
- [ ] 无未说明的跳步
- [ ] 已完成验证或说明未验证原因

### 6. 交付声明格式

```text
【交付检查】
改动文件：[文件列表]
验证命令：[命令列表 / 未运行及原因]
验证结果：[通过 / 失败 / 部分通过]
规范执行：[任务路由 / 必读规范 / skill 流程 / 无未说明跳步]
TASK_MEMORY.md：[已更新 / 不适用]
AI_MEMORY.md：[已更新 / 不适用]
遗留事项：[无 / 列出]
```

---

## 会话摘要格式

若工具支持自动保存 session summary，按以下格式：

```markdown
## 会话摘要 [YYYY-MM-DD HH:mm]

- **任务目标**：一句话描述
- **最终模式**：完整模式 / 轻量模式
- **关键决策**：选了什么方案、排除了什么
- **改动文件**：文件列表
- **验证结果**：通过 / 失败 / 部分通过（说明）
- **遗留风险**：未验证项、待确认项
- **下一步**：如有
```

---

## 禁止行为

- ❌ 未更新记忆文件就直接结束会话
- ❌ 遗留调试代码不清理也不说明
- ❌ 用"应该可以了"代替实际验证结果
- ❌ 未告知用户遗留事项或风险
