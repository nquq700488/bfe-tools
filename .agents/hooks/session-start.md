# Hook: 会话启动（Session Start）

> **模式**：`manual-check`（工具不支持真 hook 自动触发时，Agent 手动执行本清单）
>
> 触发时机：新任务会话开始，Agent 首次接收用户指令前。
> 作用：强制完成启动检查，防止跳过规范直接编码。

---

## 强制自检清单

```text
【会话启动检查】
✅/❌ 已读取 AGENTS.md：[是/否]
✅/❌ 已读取 workflow.md（工作方式与执行流程）：[是/否]
✅/❌ 已完成任务分流判断：[是/否]
✅/❌ 已完成模式判断：[是/否]
当前模式：[完整模式 / 轻量模式 / 直接回复 / 待判断]
```

### P0 阻断项（必须先解决再行动）

| 阻断项 | 处理动作 |
|--------|---------|
| AGENTS.md 未读取 | 立即读取 |
| workflow.md 未读取 | 立即读取 |
| 模式未判断 | 按 execution-mode-guidelines.md 判断 |

### P1 按需项

| 检查项 | 条件 | 处理动作 |
|--------|------|---------|
| TASK_MEMORY.md | 用户明确要求恢复上下文 / 长任务跨会话 / 上下文压缩后 | 读取 |
| AI_MEMORY.md | 涉及架构决策 / 历史坑位 / 关键链路 | 读取 |
| 专用规范 | 任务命中对应领域（编码读 coding-standards、提交读 git-workflow 等） | 按 task-routing.md 补读 |

---

## 启动声明输出

所有 P0 项满足后，输出启动声明：

```text
【会话启动检查】
✅ 已读取 AGENTS.md：是
✅ 已读取 workflow.md：是
✅ 已完成任务分流：是
✅ 已完成模式判断：是
当前模式：[完整模式 / 轻量模式 / 直接回复]
```

随后输出任务路由：

```text
【任务路由】
模式：[完整模式 / 轻量模式 / 直接回复]
使用 Skill：[无 / skill-name]
必读规范：[文件列表]
是否需要 .changes：[是/否]
下一步：[一句话说明]
```

---

## 禁止行为

- ❌ 未读取 AGENTS.md 直接回应用户编码请求
- ❌ 未判断模式直接进入编码
- ❌ 跳过 workflow.md 的任务分流步骤
- ❌ 凭记忆猜测规范内容而不重新读取
