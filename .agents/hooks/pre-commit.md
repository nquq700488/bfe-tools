# Hook: 提交前检查（Pre Commit）

> **模式**：`manual-check`（工具不支持真 hook 自动触发时，Agent 执行提交前自检）
>
> 触发时机：执行 `git commit` 前。
> 作用：强制执行质量门禁，禁止绕过验证的提交。

---

## 强制质量门禁

### 1. 变更拆分检查

- [ ] 单次提交只包含一个逻辑变更
- [ ] 不同目的的变更已拆分为多个提交
- [ ] 按依赖顺序：类型定义 / 底层逻辑 → 上层组件 → 测试 / 文档

### 2. 代码质量检查

- [ ] 无 `console.log` 残留
- [ ] 无 mock 数据混入
- [ ] 无临时调试代码
- [ ] 无未使用的导入/变量

### 3. 验证检查

| 验证项 | 标准 | 阻断？ |
|--------|------|--------|
| lint | 无 error | 是 |
| 类型检查 | 通过 | 是 |
| 必要测试 | 通过（或显式标注豁免原因） | 是 |
| 构建 | 通过（如项目有构建步骤） | 是 |

### 4. 提交信息检查

- [ ] 遵循 Conventional Commits：`type: 中文描述`
- [ ] 描述 ≤ 50 字
- [ ] type 正确（feat/fix/refactor/docs/test/chore/perf/ci）

---

## 危险操作确认

以下操作**必须询问用户确认**：

- `git reset --hard`
- `git push --force`
- `git rm`
- `--no-verify` 跳过 pre-commit hook
- 删除他人提交的代码

---

## 禁止行为

- ❌ `git add .` 不加选择地全量暂存
- ❌ `--no-verify` 绕过质量门禁
- ❌ 提交信息用英文（项目约定用中文）
- ❌ 一个大提交混装多个不相关变更
- ❌ 提交未经验证的代码
