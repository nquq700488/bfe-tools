# Hook: 编辑前检查（Pre Edit）

> **模式**：`manual-check`（工具不支持真 hook 自动触发时，Agent 在每次编辑前自检）
>
> 触发时机：修改任何文件前。
> 作用：强制执行"先读后写"，防止未读文件就修改、越界修改。

---

## 强制自检清单

### 1. 已读取确认

- [ ] 本次修改的文件**已完整读取**当前内容（不能凭印象）
- [ ] 若文件超过 500 行，至少读取修改相关段落 + 文件头/尾
- [ ] 涉及多个文件时，所有文件都已读取

### 2. 修改范围确认

- [ ] 修改范围与用户授权一致
- [ ] 不涉及计划外的文件（发现旁侧问题 → 记录但不顺手修）
- [ ] 不删除或回退他人的已有改动

### 3. 类型/接口确认

- [ ] 若涉及类型定义改动，已扫描所有引用方
- [ ] 若新增公共接口，已确认命名不与现有冲突
- [ ] 若修改函数签名，已确认调用方同步更新

---

## 快速决策树

```text
要编辑文件 X
  ↓
是否已读取 X 的当前内容？
  ├─ 否 → 先读取，再编辑
  └─ 是 → 修改范围是否超出授权？
       ├─ 是 → 记录旁侧问题，只改授权范围
       └─ 否 → 是否涉及类型/接口变更？
            ├─ 是 → 扫描引用方后再编辑
            └─ 否 → 正常编辑
```

---

## 禁止行为

- ❌ 未读取文件就直接 Edit / StrReplaceFile
- ❌ 凭"印象中的内容"修改
- ❌ 顺手修复计划外的问题（另开任务）
- ❌ 删除或回退他人已有改动（除非明确授权重构）
